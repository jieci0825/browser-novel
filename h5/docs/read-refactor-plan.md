# 移动端阅读器重构方案

## 一、需求概述

将当前基于浏览器原生滚动的 Web 式阅读器，改造为符合移动端阅读习惯的小说阅读器。

### 1.1 阅读模式

支持两种阅读模式，用户可在设置中切换：

| 模式 | 说明 | 优先级 |
|------|------|--------|
| **左右分页** | 章节文本按屏幕尺寸分页，左右滑动/点击翻页 | 主要模式 |
| **上下滚动** | 垂直方向连续滚动，章节间无缝拼接 | 次要模式 |

### 1.2 翻页动画（仅左右分页模式）

| 动画类型 | 说明 |
|---------|------|
| **平移** | 当前页与下一页一起水平滑动 |
| **覆盖** | 新页面从一侧滑入，覆盖在旧页面上方 |
| **仿真** | 模拟真实纸张翻页的卷曲效果（Canvas 实现） |
| **无动画** | 直接切换，无过渡 |

### 1.3 章节无缝衔接

两种模式均需实现：

- **向后**：读到当前章节末尾后，继续操作自动进入下一章第一页/顶部
- **向前**：读到当前章节开头时，继续操作自动进入上一章最后一页/底部
- 切换过程对用户无感，不出现加载断裂

### 2.2 文本分页引擎（PaginationEngine）

**职责**：将一章长文本按当前屏幕尺寸和排版参数分割为若干页。

**选型**：DOM 隐藏容器测量。

**原因**：

- 浏览器排版引擎自动处理 CJK 标点避头尾、字间距、行高等排版细节，无需手动实现
- 实现复杂度低，适合快速落地
- 性能足够（典型章节 20~50 段，单次测量 5~15ms）

#### 2.2.1 测量容器

在页面中创建一个不可见的测量容器，CSS 属性与实际阅读区域完全一致：

```
属性要求：
- position: absolute
- visibility: hidden
- width: 阅读区域内容宽度（viewport 宽度 - 左右 padding）
- font-size / font-family / line-height / letter-spacing / paragraph-spacing：与当前阅读设置同步
- text-indent: 2em（段首缩进）
- 不设固定 height，让内容自然撑开
```

#### 2.2.2 分页流程

```
输入：
  - paragraphs: string[]          （章节段落数组）
  - pageHeight: number            （单页可用高度 = viewport 高度 - 上下 padding）
  - 当前排版参数（字号、行高、字距、段距）

输出：
  - pages: Page[]
    Page = { paragraphs: ParagraphSlice[] }
    ParagraphSlice = {
      paragraphIndex: number       （原始段落索引）
      text: string                 （该分片的文本内容）
      isPartial: boolean           （是否为段落的部分内容，即段内被分页截断）
    }

流程：
  1. 将所有段落以 <p> 元素插入测量容器
  2. 批量读取每个 <p> 的 offsetTop 和 offsetHeight
  3. 遍历段落，按 pageHeight 累加切分：
     a. 如果当前段落底部（offsetTop + offsetHeight）未超出当前页边界 → 整段归入当前页
     b. 如果超出 → 判断段落是否需要跨页（段落高度 > 剩余空间）
        - 若段落整体高度 ≤ pageHeight：整段移到下一页
        - 若段落整体高度 > pageHeight：需要段内按行切分（见 2.2.3）
     c. 新开一页，继续累加
  4. 返回分页结果
```

#### 2.2.3 段内按行切分（处理超长段落）

当单个段落高度超过一页时，需要按行拆分：

```
1. 获取该 <p> 元素的文本节点
2. 创建 Range，从第一个字符开始，逐字符扩展 Range 范围
3. 通过 Range.getClientRects() 检测行边界：
   当 rect 的 top 值发生跳变时，说明发生了换行
4. 记录每一行的字符起止位置和高度
5. 按行累加高度，确定在该段落内的分页切分点
```

#### 2.2.4 分页缓存

```
缓存键：sourceId + bookId + chapterId + 排版参数哈希（字号+行高+字距+段距+字体+容器宽高）

缓存策略：
- 当前章 + 前一章 + 后一章的分页结果保留在内存中
- 排版参数变化时，清空所有缓存并重新分页当前章
- 章节距离超过 ±1 的缓存可释放
```

### 2.3 左右分页模式（PagedReader）

#### 2.3.1 页面结构

维护三个 DOM 槽位用于渲染：

```
┌──────────────────────────────┐
│    prevPage (上一页，隐藏)     │
│    currentPage (当前页，可见)  │
│    nextPage (下一页，隐藏)     │
└──────────────────────────────┘
```

翻页完成后，三个槽位做一次轮转复用，避免频繁创建/销毁 DOM。

#### 2.3.2 翻页交互

```
触发方式：
  - 滑动手势：touchstart → touchmove → touchend，识别水平滑动方向
  - 点击区域：屏幕左侧 1/3 区域 = 上一页，右侧 1/3 区域 = 下一页，中间 1/3 = 显示/隐藏工具栏

手势判定：
  - 水平滑动距离 > 30px 且 水平距离 > 垂直距离 → 判定为翻页手势
  - 滑动距离 < 30px 且无明显方向 → 判定为点击
```

#### 2.3.3 翻页动画实现

**平移（slide）**

```
过渡期间：
  currentPage: transform: translateX(0 → ±pageWidth)
  nextPage:    transform: translateX(±pageWidth → 0)
跟手：translateX 实时跟随 touchmove 的水平偏移
释放：根据偏移量和速度判定是否完成翻页，用 CSS transition 过渡到终态
```

**覆盖（cover）**

```
向后翻：
  currentPage: 不动
  nextPage:    transform: translateX(pageWidth → 0)，层级在上，附带阴影
向前翻：
  currentPage: transform: translateX(0 → pageWidth)，层级在上，附带阴影
  nextPage:    不动（或轻微反向位移模拟层叠感）
```

**仿真（simulation）**

```
实现方式：Canvas 绘制
1. 将 currentPage 和 nextPage 的 DOM 通过离屏渲染快照为位图
2. 根据手指触摸点计算翻页卷曲的贝塞尔曲线控制点
3. 在 Canvas 上绘制：
   - 底层：下一页位图
   - 卷曲区域：当前页位图经过几何变换后的弯曲效果
   - 阴影：沿卷曲边缘的渐变阴影
4. 用 requestAnimationFrame 驱动动画
```

**无动画（none）**

```
直接替换 currentPage 的内容，无过渡。
```

#### 2.3.4 页面快照方案（仿真翻页用）

由于仿真翻页需要页面的位图数据：

```
方案：离屏 Canvas 绘制 DOM 内容
1. 创建一个与页面同尺寸的离屏 Canvas
2. 将页面 DOM 的内容（纯文本段落）用 Canvas API 重新绘制到离屏 Canvas 上
   - 按分页引擎的分页结果，逐段逐行绘制文字
   - 使用 ctx.fillText 渲染每一行
3. 该 Canvas 即可作为仿真翻页动画的输入位图

为什么不用 html2canvas：
  - 依赖体积大
  - 移动端性能差
  - 对于纯文本场景，直接用 Canvas fillText 绘制更轻量、更可控
```

### 2.4 上下滚动模式（ScrollReader）

#### 2.4.1 核心结构

```
┌────────────────────────────────┐
│  chapter N-1 的内容（按需加载）   │
│  chapter N 的内容（当前章节）     │
│  chapter N+1 的内容（按需加载）   │
└────────────────────────────────┘
整体为一个可滚动容器，使用原生滚动
```

#### 2.4.2 无缝章节加载

```
向下滚动：
  1. 监听 scroll 事件（throttle / IntersectionObserver）
  2. 当距离底部 < 阈值（如 500px）时，加载下一章内容并 append 到容器底部
  3. 加载完成后，滚动位置自然连续，无跳变

向上滚动：
  1. 当距离顶部 < 阈值时，加载上一章内容并 prepend 到容器顶部
  2. prepend 后，需要修正 scrollTop 以保持视觉位置不跳变：
     scrollTop += 新插入内容的高度

内存管理：
  - 当已加载的章节数 > 5 时，移除距离当前最远的章节 DOM
  - 移除时同样修正 scrollTop
```

#### 2.4.3 当前章节识别

```
在滚动过程中，需要实时判定用户当前正在阅读哪一章：
- 每个章节容器标记 data-chapter-id
- 通过 IntersectionObserver 或滚动位置计算，判定视口中占比最大的章节为当前章节
- 当前章节变化时，更新路由参数（replace，不产生历史记录）和阅读进度
```

### 2.5 章节预加载（ChapterPreloader）

```
职责：提前加载相邻章节数据，供两种阅读模式使用。

缓存池：
  - 数据结构：Map<chapterId, { content: ChapterContent, pages?: Page[] }>
  - 容量：当前章 + 前后各 1 章（共 3 章），超出范围的自动清理

预加载时机：
  - 左右分页模式：翻到当前章节的最后 3 页或前 3 页时，触发相邻章节预加载
  - 上下滚动模式：距离章节边界 < 阈值时触发（与 2.4.2 的加载逻辑合并）

加载流程：
  1. 请求章节内容（API）
  2. 缓存内容数据
  3. 若当前为分页模式，在 requestIdleCallback 中执行分页计算并缓存结果
```

## 三、边界处理

### 3.1 分页相关

| 场景 | 处理方式 |
|------|---------|
| **排版参数变化时的阅读位置保持** | 分页前记录当前页首段落索引 + 段内字符偏移量，重新分页后根据该偏移量定位到对应新页码 |
| **单段落超过一页** | 进入段内按行切分逻辑（2.2.3），将超长段落拆为多个 ParagraphSlice 分属不同页 |
| **空章节（无内容）** | 显示一页占位内容（如"本章暂无内容"），页数为 1 |
| **章节内容含连续空行** | 分页前过滤空行，与当前 `content.split('\n').filter(Boolean)` 逻辑一致 |
| **容器尺寸变化（屏幕旋转等）** | 监听 resize 事件（debounce 300ms），触发重新分页 + 位置保持 |

### 3.2 章节切换相关

| 场景 | 处理方式 |
|------|---------|
| **已到第一章，继续向前翻** | 不响应手势，给出轻提示（如短暂回弹 + toast"已经是第一章了"） |
| **已到最后一章，继续向后翻** | 同上，提示"已经是最后一章了" |
| **下一章数据尚未加载完成** | 显示一页 loading 占位（带骨架屏），加载完成后自动替换为真实内容并计算分页 |
| **章节加载失败** | 显示错误页（带重试按钮），不影响当前章节的正常阅读 |
| **快速连续翻页穿越多章** | 取消正在进行的旧请求（AbortController），只保留最新目标章节的请求 |

### 3.3 手势与动画相关

| 场景 | 处理方式 |
|------|---------|
| **翻页手势与垂直滚动冲突** | 判定滑动方向：水平位移 > 垂直位移时拦截为翻页手势，否则放行（分页模式下不应有垂直滚动） |
| **翻页动画进行中再次触发翻页** | 忽略新手势，直到当前动画完成。或者快速完成当前动画后再响应新手势 |
| **手势滑动距离不足** | 回弹到原位，不触发翻页。阈值：滑动距离 < 页面宽度的 1/3 且滑动速度 < 阈值 |
| **仿真翻页的性能保护** | 限制 Canvas 绘制帧率与页面位图尺寸（可降采样），低端设备检测后自动降级为覆盖或平移动画 |

### 3.4 模式切换相关

| 场景 | 处理方式 |
|------|---------|
| **从分页切换到滚动模式** | 根据当前页码计算对应段落位置，切换后滚动到该段落 |
| **从滚动切换到分页模式** | 根据当前 scrollTop 判定可见段落，执行分页后定位到包含该段落的页码 |
| **切换模式时章节不变** | 不重新请求数据，复用已缓存的章节内容 |

### 3.5 阅读进度相关

| 场景 | 处理方式 |
|------|---------|
| **分页模式下的进度记录** | 记录 chapterId + pageIndex，恢复时根据 pageIndex 直接跳转 |
| **滚动模式下的进度记录** | 记录 chapterId + scrollTop（或段落索引 + 段内偏移比例），恢复时滚动到对应位置 |
| **排版参数变化导致 pageIndex 失效** | 进度中同时存储段落索引 + 字符偏移量作为备用定位依据 |
| **目录跳转** | 清空预加载缓存中与新位置无关的数据，加载目标章节，定位到第一页/顶部 |
